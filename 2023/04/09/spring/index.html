Spring简介

1. Spring使用的策略

  • 基于POJO的轻量级和最小侵入式编程；
  • 通过控制反转和依赖注入（对象管理）以及面向接口（JDBC）实现松耦合；
  • 基于切面和惯例进行声明式编程；
  • 通过切面和模板减少样板式代码。

2. 相关模块

  • Spring核心模块：

    • spring-core: IOC和DI的基本实现。
    • spring-bean: Bean工厂和bean的装配。
    • spring-context: 定义基础的context上下文和IOC容器。
    • spring-context-support: springIOC的扩展支持以及子容器。
    • spring-expression: spring表达式语言。

  • Spring面向切面模块：

    • spring-aop: 面向切面编程的应用模块。
    • spring-aspects: 整合Aspectj、AOP模块。
    • spring-instrument: 动态Class Loading模块。

  • Spring数据访问和集成模块。

    • spring-jdbc: 提供JDBC抽象框架的主要实现模块（面向接口编程）。
    • spring-tx: JDBC实现事务管理模块。
    • spring-orm: 对象关系映射。
    • spring-oxm:对象XML映射。
    • spring-jms: Java message service接受发送消息模块。

  • Spring的web模块：

    • spring-web: 提供最基本的web支持，建立在核心容器上，通过servlet或者Listener初始化IOC容器。
    • spring-webmvc: 实现Spring MVC的web应用。
    • spring-websocket: 实现与web前端的双全工通讯的协议（前后端可以互相发送请求）。

  • Spring测试和通信报文模块。

    • spring-test: 为测试提供支持。
    • spring-messaging: 为Spring框架集成一些基础的报文传送应用。

  • Spring结构图和各模块之间的依赖。

3. Spring IOC

  • IOC概念：

    IOC是中思想，即把对对象的控制权从程序本身转移给外部容器，由容器控制对象的生命周期。

  • DI控制注入：

    当前实现IOC的最优秀的解耦方式，将bean通过配置文件的方式耦合在一起。

  • XML的方式实现IOC

    • XML文件配置：

      底层实现：

      clazz = class.forName(class)

      Controuctor c = class.getDeclareContorctor()

      c.newInstance()

    • 调用方式：

      • 方式一：

        //创建IoC容器对象的时候就会初始化容器中管理的所有的Bean

        ApplicationContext ac =

        new ClassPathXmlApplicationContext(“applicationContext.xml”);

        // 从容器中获取User对象

        User user = (User)ac.getBean(“user”);

      • 方式二：

        // 创建IoC容器对象，BeanFactory仅仅只会加载解析配置文件，并不会创建具体的实例对象

        BeanFactory bf =

        new XmlBeanFactory(newClassPathResource(“applicationContext.xml”));

        // 去容器中根据条件获取对象，如果对象没有实例化那么就会创建该对象

        User user = (User)bf.getBean(“stu1”,Student.class);

      • getBean()

        该方法默认调用类的无参构造方法，如果构造方法被重载，会报错。

        getBean参数：id、name、class。

        根据name获取时，bean和名字时一对多的关系。id=”1,2,3”，则该Bean有1，2，3三个名字。

        根据id获取时，bean和名字时一对一的关系。id=”1,2,3”，则该Bean的名字就是1,2,3。

        根据class获取时，如果该class有多个bean则会报错，此时可以通过class+id的组合方式获取。

    • IOC对象工厂注入

      应对容器需要创建管理的对象过多时的情况。将对象的创建交由第三方工厂。

      • 静态工厂

        创建工厂类，定义实例化对象的静态方法：

        public static User getInstance(){

        ​ return new User();

        }

        在XML中配置静态工厂：

        <bean class=”com.bobo.ioc.factory.UserFactory” factory-method=”getInstance”

        id=”user” />

        此时因为getInstance()方法是静态的，可以直接通过class调用。

      • 动态工厂

        创建工厂类，定义实例化对象的静态方法：

        public User getInstance(){

        ​ return new User();

        }

        在XML中配置动态工厂：

        <bean class=”com.bobo.ioc.factory.UserDynamicFactory”

        id=”userDynamicFactory” />

        <bean id=”user” factory-bean=”userDynamicFactory” factory

        method=”getInstance” />

        此时需要注册动态工厂bean，然后根据工厂对象调用getInstance()方法。
