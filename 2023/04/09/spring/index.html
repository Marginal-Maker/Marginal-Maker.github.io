Spring简介

1. Spring使用的策略

  • 基于POJO的轻量级和最小侵入式编程；
  • 通过控制反转和依赖注入（对象管理）以及面向接口（JDBC）实现松耦合；
  • 基于切面和惯例进行声明式编程；
  • 通过切面和模板减少样板式代码。

2. 相关模块

  • Spring核心模块：

    • spring-core: IOC和DI的基本实现。
    • spring-bean: Bean工厂和bean的装配。
    • spring-context: 定义基础的context上下文和IOC容器。
    • spring-context-support: springIOC的扩展支持以及子容器。
    • spring-expression: spring表达式语言。

  • Spring面向切面模块：

    • spring-aop: 面向切面编程的应用模块。
    • spring-aspects: 整合Aspectj、AOP模块。
    • spring-instrument: 动态Class Loading模块。

  • Spring数据访问和集成模块。

    • spring-jdbc: 提供JDBC抽象框架的主要实现模块（面向接口编程）。
    • spring-tx: JDBC实现事务管理模块。
    • spring-orm: 对象关系映射。
    • spring-oxm:对象XML映射。
    • spring-jms: Java message service接受发送消息模块。

  • Spring的web模块：

    • spring-web: 提供最基本的web支持，建立在核心容器上，通过servlet或者Listener初始化IOC容器。
    • spring-webmvc: 实现Spring MVC的web应用。
    • spring-websocket: 实现与web前端的双全工通讯的协议（前后端可以互相发送请求）。

  • Spring测试和通信报文模块。

    • spring-test: 为测试提供支持。
    • spring-messaging: 为Spring框架集成一些基础的报文传送应用。

  • Spring结构图和各模块之间的依赖。

  • 基于注解的形式

    • 添加SpringAOP依赖，AOP中包含了注解的实现。

    • 使用注解方式时，需要添加扫描路径，指定Spring容器需要加载的Java类。

    • @Component注解：

      被该注解标识的Java类会被Spring容器加载。

    • @Resource注解和@Autowired注解

      在注解的场景下依赖注入我们可以通过 @Resource和@Autowired实现，他们的区别是@Resource

      默认是根据name属性来查找的，而@Autowired注解默认是根据类型查找的，@Autowired需要和

      @Qualifier配合实现基于name的查找。

    • @Component相关注解分类：

      注解 描述
      @Component 一般用在身份不明确的组件上
      @Service 一般用在Service层
      @Controller 一般用在Controller层
      @Repository 一般用在Dao层

    • 设置扫描时的特殊条件，比如在SSM框架中Spring基础配置文件只需要扫描Dao和Service层，不需要Controller层。而SpringMVC配置文件中只需要扫描Controller层。
